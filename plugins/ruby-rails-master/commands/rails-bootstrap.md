# /rails-bootstrap

Bootstrap a new Rails app with the conventions this plugin teaches: opinionated Gemfile, service / form / query / value object directories, rubocop config, pre-commit hooks, factory_bot, basic CI.

## What to do

1. **Confirm intent.** Confirm with the user:
   - App name (`$ARGUMENTS` if provided).
   - Test framework (Minitest default; RSpec optional).
   - DB (PostgreSQL default).
   - Deploy target (Kamal, Render, Heroku, custom).
   - Frontend approach (Hotwire default; Tailwind + esbuild).

2. **Run the Rails generator** with opinionated flags. Show the user the command before running:
   ```bash
   rails new <app-name> \
     --database=postgresql \
     --css=tailwind \
     --javascript=esbuild \
     --skip-jbuilder \
     [--minimal | --skip-test if RSpec]
   ```

3. **Spawn the `rails-architect` subagent** to produce the architecture document.

4. **Apply the bootstrap layer.** This includes:

   ### Directories
   ```
   app/services/
   app/services/.keep
   app/queries/
   app/forms/
   app/policies/        # if using Pundit
   app/values/
   app/serializers/     # if using Alba/Jbuilder/etc.
   ```

   ### Templates from the plugin
   Copy `templates/*.rb` examples into `app/services/`, `app/queries/`, etc., as starting points. Mark them `.example` so they don't bootstrap as live code.

   ### Configuration files
   - `templates/rubocop.yml` → `.rubocop.yml`
   - `templates/pre-commit-config.yaml` → `.pre-commit-config.yaml`

   ### Gemfile additions (after `rails new` completes)
   ```ruby
   group :development, :test do
     gem "rubocop", require: false
     gem "rubocop-rails", require: false
     gem "rubocop-performance", require: false
     gem "brakeman", require: false
     gem "bundler-audit", require: false
     gem "bullet"
     gem "strong_migrations"
     gem "factory_bot_rails"
     gem "faker"
     # if RSpec
     # gem "rspec-rails"
     # gem "shoulda-matchers"
   end

   group :development do
     gem "annotate"
     gem "letter_opener"
     gem "rack-mini-profiler"
   end
   ```

   ### Initializers
   - `config/initializers/bullet.rb` — N+1 detection in dev.
   - `config/initializers/strong_migrations.rb` — guard rails for migrations.

   ### CI
   - `.github/workflows/ci.yml` running `bundle exec rubocop`, `bundle exec brakeman`, `bin/rails db:test:prepare`, `bin/rails test` (or `bundle exec rspec`).

   ### Documentation
   - `CLAUDE.md` at the project root — describes the conventions Claude should follow when working in this repo.

5. **Show the user the project tree** and explain what each non-Rails-default piece does. Note what they'll want to customize before their first PR.

## Behavior notes

- Don't run irreversible operations without confirming. `rails new` is fine; gem installs are fine; deleting existing files needs confirmation.
- Default to running in a sandbox (`/tmp/<app-name>` or a worktree) if the user hasn't picked a directory.
- For an existing project, use `/find-smells` and `/rails-review` instead — bootstrap is for greenfield.

## When to refuse

- "Bootstrap into an existing project" — the bootstrap is destructive (overwrites `.rubocop.yml` etc.). Recommend doing a partial application: `cp templates/rubocop.yml .rubocop.yml`, etc.
- "Use Mongo/no DB" — challenge first; if the user genuinely needs it, swap the `--database` flag.
- "Skip tests" — refuse. Tests are part of the bootstrap.
